# Практична робота: Реалізація принципів SOLID
**Виконала:** Шмиголь Олександра

## Аналіз порушень у вихідному коді (src/original)
1. **SRP**: Клас `OrderManager` одночасно рахував ціну, зберігав у БД та надсилав SMS.
2. **OCP**: Логіка була жорстко прописана, що заважало додавати нові типи сповіщень.
3. **DIP**: Клас залежав від конкретних реалізацій, а не від абстракцій.

## Опис проведеного рефакторингу (src/refactored)
- **SRP (Single Responsibility)**: Розділено логіку обробки замовлення (`OrderProcessor`) та сповіщення (`SmsNotification`).
- **ISP (Interface Segregation)**: Створено вузькоспеціалізований інтерфейс `INotification`.
- **DIP (Dependency Inversion)**: `OrderProcessor` тепер залежить від інтерфейсу `INotification`. Це дозволяє легко замінити SMS на Email, не змінюючи код процесора.
